# math_snake
Создание классической игры "Змейка" на разных топологических поверхностях
# Змейка на топологических поверхностях

Классическая «Змейка» в терминале, но игровое поле — не плоскость, а
поверхность, полученная **склейкой краёв квадрата**. Один и тот же код по-разному
обрабатывает выход за границу в зависимости от выбранной топологии.

Поддерживаются шесть случаев: плоскость, цилиндр, тор, лента Мёбиуса,
бутылка Клейна и проективная плоскость **RP²**. Переключаются прямо во время
игры (клавиши `1`–`6`), что позволяет наглядно увидеть разницу.

---

## Математическая идея

Поле размером `W × H` — это фундаментальный многоугольник (квадрат). Тип
поверхности задаётся тем, как отождествлены его противоположные стороны.
В терминах слов на сторонах `a` (горизонталь) и `b` (вертикаль):

| Поверхность            | Гор. края (лево↔право)      | Верт. края (верх↔низ)       | Слово        | Ориентируемость |
|------------------------|-----------------------------|-----------------------------|--------------|-----------------|
| Плоскость              | стены                       | стены                       | —            | —               |
| Цилиндр                | склейка напрямую            | стены                       | `a b a⁻¹`    | да              |
| Тор                    | склейка напрямую            | склейка напрямую            | `a b a⁻¹ b⁻¹`| да              |
| Лента Мёбиуса          | склейка с переворотом       | стены                       | —            | **нет**         |
| Бутылка Клейна         | склейка с переворотом       | склейка напрямую            | `a b a b⁻¹`  | **нет**         |
| Проективная плоскость  | склейка с переворотом       | склейка с переворотом       | `a b a b`    | **нет**         |

«Склейка с переворотом» означает, что край отождествляется с противоположным
с отражением: например, при выходе через правый край в точке высоты `y` змейка
появляется слева на высоте `H-1-y`. Именно это отражение делает поверхности
Мёбиуса, Клейна и RP² **неориентируемыми**.

### Тонкость: преобразование направления

Отображение склейки переносит не только позицию, но и **вектор скорости**
(через свой якобиан: `diag(1,-1)` для вертикального шва, `diag(-1,1)` для
горизонтального). Меняет знак только компонента скорости *вдоль* шва.

Поскольку змейка ходит строго по осям единичными шагами, в момент пересечения
шва скорость всегда **перпендикулярна** ему, а вдоль-шовная компонента равна
нулю. Поэтому переворот направления для этой игры тривиален — значимым
остаётся перенос позиции. Формула якобиана оставлена в коде ради
математической корректности (и проверена тестом на диагональном векторе).

## Сборка и запуск

Требуется компилятор с поддержкой C++17 (GCC/Clang). Платформа — POSIX
(Linux / macOS): используется `termios` для «сырого» неблокирующего ввода.

### Через CMake

```bash
cmake -S . -B build
cmake --build build
./build/snake          # игра
ctest --test-dir build # тесты топологии
```

### Напрямую через g++

```bash
g++ -std=c++17 -O2 -Isrc src/*.cpp -o snake && ./snake
```

## Управление

| Клавиши            | Действие                                  |
|--------------------|-------------------------------------------|
| `W A S D` / стрелки| движение                                  |
| `1` … `6`          | сменить поверхность на лету               |
| `R`                | начать заново                             |
| `Q` / `Ctrl-C`     | выход                                     |

Обозначения: `@` — голова, `o` — тело, `*` — еда.

## Структура проекта

```
topo-snake/
├── CMakeLists.txt
├── README.md
├── src/
│   ├── vec2.h         # целочисленная 2D-точка/вектор
│   ├── topology.h     # перечисление топологий + объявление wrap()
│   ├── topology.cpp   # правила склейки краёв (вся «топология»)
│   ├── game.h         # интерфейс игровой логики
│   ├── game.cpp       # состояние, движение, еда, столкновения
│   ├── terminal.h     # RAII raw-режим терминала
│   ├── terminal.cpp   # неблокирующий ввод + отрисовка ANSI
│   └── main.cpp       # игровой цикл
└── tests/
    └── topology_test.cpp  # проверка склеек (assert, без терминала)
```

Логика, ввод-вывод и математика разделены: `topology.*` ничего не знает об
игре, `game.*` — о терминале. Это упрощает тестирование (тест собирается из
одного `topology.cpp`) и позволяет заменить отрисовку, не трогая правила.
